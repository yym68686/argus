# Design Context

## Product
Argus is a self-hosted operator console for managing gateway sessions, user fleets, upstream channels, usage, and runtime health.

## Users
- Technical operators
- Self-hosters
- Telegram-first power users
- Engineers inspecting live agent/runtime activity

## Brand Personality
- Precise
- Trustworthy
- Quiet
- Operator-grade
- High-signal, low-noise

## Visual Direction
- Dark control-room interface, not glossy consumer SaaS
- Sans-serif typography with disciplined mono accents for identifiers and telemetry
- One cool accent family only; avoid purple/cyan AI clichés
- Strong hierarchy, restrained decoration, visible state
- Surfaces should feel layered and infrastructural, not like interchangeable cards

## UX Principles
- Put system state first
- Keep filters and actions close to the data they affect
- Reduce repeated border-box patterns when grouping can do the job
- Empty, loading, and degraded states should still feel designed
- Data-dense views should stay calm and readable on wide screens
